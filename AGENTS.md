# Agent Handoff Notes

## Project

This repo is a Create React App site for a mobile-first birthday-month experience called **Celebrating Jessica**.

The site is deployed to GitHub Pages at:

```txt
https://qthibodeaux.github.io/may
```

Core concept: a 17-day personalized advent-calendar/scrapbook path for Jessica from **May 1-17, 2026**, ending on **Birthday Queen Day**.

## Current Structure

Important files:

```txt
src/App.js
src/App.css
src/content/days.js
src/content/story.js
public/index.html

```

Content is intentionally kept in JS files so the user can update ideas quickly without touching most UI code.

## Routing

The app uses hash routes for GitHub Pages compatibility.

Examples:

```txt
/#/
/#/day/1
/#/story
/#/story/1
/#/memories
```

Do not switch to browser history routing unless GitHub Pages fallback handling is also added.

## Date Locking

Days unlock visually by date using `America/Chicago`.

Current date behavior lives in `src/App.js`.

There is also a local preview override:

```txt
?today=2026-05-17
```

Example:

```txt
http://localhost:3000/may?today=2026-05-17
```

This is visual hiding only. The page is public and static, so content is not truly secure.

## No Direct useEffect

The user explicitly does not want direct `useEffect` usage.

There is a doctrine file one directory up:

```txt
../useEffectDoctorine.md
```

Current route subscription uses `useSyncExternalStore`, not `useEffect`. Keep it that way. Prefer derived state, event handlers, and explicit components. If a true mount-only external sync is needed later, discuss it first or use a named abstraction rather than direct component-level `useEffect`.

## Design Direction

Current visual direction:

```txt
black and gold
mobile-first
playful family photo album
premium scrapbook / invitation feel
```

Avoid turning it into a generic party flyer. Keep it personal, warm, and polished.

Planned days currently marked with `hasPlan: true`:


Locked planned days show a subtle anticipatory marker and `Something is waiting` without revealing the title/content.

## Riddles

Riddles are rendered by `RiddleReveal` in `src/App.js`.

Riddle answers are normalized by lowercasing and stripping punctuation/spaces. This means guesses like `Torchy's Tacos`, `TORCHYS TACOS`, and `torchys` can all work if listed in `answers`.

Basic riddle shape:

```js
riddle: {
  question: '...',
  clues: ['...', '...'],
  answers: ['picnic', 'family picnic'],
  successTitle: 'You got it.',
  successMessage: '...',
  revealUrl: 'https://...',
  revealLabel: 'Open the map'
}
```

Ticket reveal shape:

```js
ticket: {
  eyebrow: 'Birthday Month Invitation',
  title: 'Family Picnic Pass',
  admit: 'Jessica + family',
  date: 'May 2, 2026',
  time: 'Today',
  location: 'Richard & Meg Weekley Park',
  address: '8440 Greenhouse Road, Cypress, TX 77433',
  note: 'Food, fresh air, a lake view, and time together.',
  image: publicAsset('/assets/PXL_20250729_200727727.jpeg'),
  imageAlt: 'Family picnic invitation photo'
}
```

## Build And Deploy

Verify changes with:

```txt
npm run build
```

Push source changes:

```txt
git add .
git commit -m "Your message"
git push origin master
```

Publish GitHub Pages:

```txt
npm run deploy
```

`npm run deploy` runs `predeploy`, which runs `npm run build`, then publishes `build` to the `gh-pages` branch.

## Local Dev

Start the local server:

```txt
npm start
```

Local URL:

```txt
http://localhost:3000/may
```

If running in this agent environment on Windows, starting the dev server may require escalation and `npm.cmd`.

## Notes For Future Agents

- Keep changes focused. This is a personal gift site, not a general product.
- Mobile experience matters most.
- Preserve hash routing.
- Do not add direct `useEffect`.
- Run `npm run build` after content edits, especially when pasting story text with quotes/apostrophes.
- Ask before pushing/deploying unless the user explicitly requests it.
